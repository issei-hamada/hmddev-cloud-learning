以下の手順で授業を開始してください。

## ステップ1: フェーズを判定する

まず `records/learning_plan.json` の存在を確認する。

- **存在する → フェーズ2**（推奨プランに従った復習）
- **存在しない → フェーズ1**（初回学習）

---

## フェーズ1: 初回学習

### 開始点の決定

`records/progress.json` を read ツールで読み込む。

```
progress.json が存在しない
  → IT初心者チェックがまだ行われていない
  → teacher エージェントの起動時の動作（IT初心者チェック）へ戻る

progress.json が存在する場合:
  → it_beginner が true で appA の status が not_started または存在しない
       → appA（ネットワーク基礎）から開始する
  → status が in_progress の章がある
       → その章の last_section_index + 1 のセクションから再開する
  → in_progress の章がない
       → 最後に completed になった章の次の章を特定して提案する
  → 全章 not_started（progress.json はあるが授業未開始）
       → it_beginner の値に応じて appA または ch00 から開始する
```

### 章の順序（フェーズ1）

appA（初心者のみ） → ch00 → ch01 → ch02 → ch03 → ch04
→ MPL-1 → ch05 → ch06 → ch07 → ch08
→ MPL-2 → ch09 → ch10 → ch11 → ch12
→ MPL-3 → ch13 → ch14 → ch15 → ch16
→ FPL

### ユーザーへの確認

開始点が決まったら、以下のように伝えてから授業を始める:

- 続きから: 「前回の続き（〇〇章 セクション△△）から始めます！」
- 新しい章: 「次は〇〇章から始めましょう！」

ユーザーが「別の章から始めたい」と言った場合は以下の一覧を提示する:

```
利用可能な章:
  付録A: 通信とインターネットの基礎（IT初心者向け）
  第0章: 導入 ― クラウドの基本概念と AWS の全体像
  第1章: Webアプリケーションのホスティング
  第2章: サーバーレスアーキテクチャ
  第3章: データの永続化 ― ストレージ戦略
  第4章: データベース設計
  第5章: ネットワーク設計
  第6章: 非同期処理とアプリケーション統合
  第7章: 耐障害性と高可用性
  第8章: コンテナ技術
  第9章: セキュリティとアクセス管理
  第10章: 監視・運用と管理
  第11章: CI/CD とデベロッパーツール
  第12章: データ分析基盤
  第13章: AI/ML サービス
  第14章: クラウド移行戦略
  第15章: コスト管理と最適化
  第16章: その他の頻出サービス
  付録B: AWS サポートプランの比較（任意）
  付録C: 試験対策のポイント（任意）
```

## フェーズ2: 推奨プランに従った復習

`records/learning_plan.json` を read ツールで読み込む。

### 次の章を特定する

`plan` 配列の中で `status: "pending"` の最初のエントリを選ぶ。

### ユーザーへの提案

「次は **〇〇章（<chapter_title>）** の復習をしましょう。

理由: <reason>

特に以下のセクションを重点的に確認します:
<focus_sections をリストで表示>

始めますか？」

### 授業の実施

teacher エージェントの授業フローを実行する。`focus_sections` に記載のセクションでは確認質問を **2問** にする。

### 章完了後の処理

授業終了 → Quick Test（5問・その章の問題）を実施。

Quick Test 終了後、`records/learning_plan.json` を write ツールで更新する:

```json
（対象エントリの status を "pending" → "completed" に変更）
（completed_reviews 配列に章ID を追記）
```

次の pending エントリへ進む。全章が completed の場合は FPL 再受験を提案する。
