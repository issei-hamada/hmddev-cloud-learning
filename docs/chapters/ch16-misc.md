## 16-1. エンドユーザーコンピューティング

### このセクションで学ぶこと

「会社のパソコン環境をクラウド上に用意する」サービスを学びます。WorkSpaces、WorkSpaces Secure Browser、AppStream 2.0 の3つを押さえ、それぞれの違いを理解することがゴールです。

### Amazon WorkSpaces ― クラウド上の仮想デスクトップ

Amazon WorkSpaces は、クラウド上に「自分専用のパソコンのデスクトップ」を用意してくれるサービスです。このようなサービスを DaaS（Desktop as a Service）と呼びます。第0章で学んだ IaaS や SaaS と同じ「〜 as a Service」の仲間で、デスクトップ環境をサービスとして提供する形態です。

イメージとしては、「会社が貸してくれるパソコンが、インターネットの向こう側にある」という感覚です。手元のパソコンやタブレットからインターネット経由で接続すると、いつもと同じWindowsやLinuxのデスクトップ画面が表示されます。ファイルもアプリケーションもすべてクラウド側にあるので、手元の端末が壊れてもデータは安全です。

第1章で学んだ EC2 との違いが気になるかもしれません。EC2 は「サーバー」です。Webサイトやアプリケーションを動かすための「裏方の機械」として使います。技術的には EC2 にリモートデスクトップ接続して操作することも可能ですが、WorkSpaces はデスクトップ用途に特化したマネージドサービス（AWSが運用管理を担ってくれるサービス）です。ユーザー管理、データの永続保存、画面転送の最適化、社内の認証システム（Active Directory）との連携などが最初から組み込まれており、「社員用の作業環境」として使うなら WorkSpaces の方が圧倒的に手軽です。

WorkSpaces がよく使われるのは、リモートワークの環境を整えたいとき、派遣社員や外部パートナーに一時的な作業環境を提供したいとき、セキュリティの観点から社員の端末にデータを残したくないときなどです。

### WorkSpaces Secure Browser

WorkSpaces Secure Browser は、名前のとおり「セキュアなWebブラウザだけをクラウドから提供する」サービスです。デスクトップ全体ではなく、ブラウザだけを提供するのが特徴です。

社内のWebアプリケーションに安全にアクセスさせたいけれど、デスクトップ環境をまるごと用意するほどではない、という場面で使います。たとえば、派遣社員が自分のパソコンから社内の勤怠管理システムだけを使えるようにしたい場合などに便利です。WorkSpaces よりも軽量で、コストも抑えられます。

### Amazon AppStream 2.0 ― アプリケーション単位のストリーミング

Amazon AppStream 2.0 は、特定のアプリケーションだけをクラウドからストリーミング配信するサービスです。

WorkSpaces が「パソコン丸ごと」をクラウドに用意するのに対して、AppStream 2.0 は「特定のアプリだけ」を配信します。たとえば、設計用のCADソフトや動画編集ソフトなど、高性能なパソコンが必要なアプリケーションを使いたいとき、社員一人ひとりに高価なパソコンを買い与える代わりに、AppStream 2.0 でクラウドから配信すればコストを抑えられます。

WorkSpaces と AppStream 2.0 の違いは試験でよく問われます。整理すると、WorkSpaces は「デスクトップ環境をまるごと提供」で、日常的な業務用パソコンの代替として使います。AppStream 2.0 は「特定のアプリケーションだけを配信」で、特定の業務ソフトを使わせたいときに使います。「デスクトップごとか、アプリ単位か」がポイントです。

---

> **試験のポイント**
> - WorkSpaces = クラウド上の仮想デスクトップ（DaaS）。リモートワークやセキュリティ対策で利用
> - AppStream 2.0 = アプリケーション単位のストリーミング配信
> - 「デスクトップ全体」なら WorkSpaces、「特定のアプリだけ」なら AppStream 2.0
> - WorkSpaces はデスクトップ用途に特化したマネージドサービス。EC2 でもリモート操作は可能だが、WorkSpaces はユーザー管理や認証連携が組み込み済み

## 16-2. IoT

### このセクションで学ぶこと

IoT（Internet of Things）の基本と、AWSでIoTデバイスをクラウドに接続するための AWS IoT Core の概要を理解します。CLF-C02 では深い知識は問われませんが、「何をするサービスか」を押さえておきましょう。

### AWS IoT Core ― IoTデバイスをクラウドにつなげる

IoT とは「モノのインターネット」と訳されます。センサー、カメラ、家電、工場の機械など、さまざまな「モノ」がインターネットにつながって、データを送受信する仕組みのことです。

AWS IoT Core は、こうしたIoTデバイス（センサーや機器など）をAWSクラウドに安全に接続し、管理するためのサービスです。たとえば、工場に設置された何千台もの温度センサーが計測データをクラウドに送り、異常を検知したらアラートを出す、といった仕組みを構築できます。

IoT Core は MQTT（エムキューティーティー）というプロトコル（通信の約束ごと）に対応しています。MQTT は、データ量が小さく電力消費も少ないため、バッテリーで動く小型のセンサーなどに適した通信方式です。試験では「MQTT」というキーワードが出たら IoT Core を思い浮かべてください。

IoT Core に届いたデータは、Lambda で処理したり、S3 に保存したり、Kinesis でリアルタイム分析したりと、他のAWSサービスと組み合わせて活用できます。なお、IoT Core にはデバイスシャドウ（デバイスがオフラインでも最後の状態を保持する機能）やルールエンジン（受信データに応じて自動的にAWSサービスを呼び出す機能）といった仕組みもありますが、CLF-C02 ではここまでの詳細は問われません。

---

> **試験のポイント**
> - IoT Core = IoTデバイスをAWSクラウドに安全に接続・管理するサービス
> - MQTT プロトコルに対応（軽量な通信方式 → IoTに適している）
> - CLF-C02 では概要レベルの理解で十分

---

## 16-3. ビジネスアプリケーション

### このセクションで学ぶこと

AWSが提供するビジネス向けアプリケーションサービスとして、Amazon Connect（コンタクトセンター）と Amazon SES（メール送信）の2つを押さえます。

### Amazon Connect ― クラウドで作るコールセンター

Amazon Connect は、コンタクトセンター（いわゆるコールセンター）をクラウド上に構築できるサービスです。

従来、コールセンターを作るには専用の電話交換機を購入し、回線を引き、ソフトウェアを導入して……と大がかりな準備が必要でした。Amazon Connect を使えば、そうした物理的な設備なしに、Webブラウザだけでコールセンターを立ち上げられます。

電話の受発信、自動音声応答（IVR：電話をかけると「1番を押してください」と案内される、あれです）、通話録音、オペレーターへの振り分けなどの機能が揃っています。使った分だけ料金がかかる仕組み（従量課金）なので、小規模なサポート窓口から大規模なコールセンターまで柔軟に対応できます。

試験では「クラウドベースのコンタクトセンター」「コールセンターをクラウドで構築」というキーワードが出たら Amazon Connect を選びましょう。

### Amazon SES ― 大量メールの送信サービス

Amazon SES（Simple Email Service）は、大量のメールを送信するためのサービスです。

たとえば、ECサイトが注文確認メールを送ったり、マーケティング部門がキャンペーンメールを10万人の顧客に一斉送信したりする場面で使います。自前のメールサーバーを構築・運用する手間がなく、高い到達率（迷惑メールとして弾かれにくい）が特徴です。

SES は「メールを送るサービス」、SNS（Simple Notification Service）は「通知を届けるサービス」です。SNS はメールだけでなくSMS（ショートメッセージ）やHTTPエンドポイントへの通知もできますが、SES はメールに特化したサービスです。送信が主な用途ですが、受信機能（受信したメールを S3 に保存したり Lambda で処理したりする）も備えています。HTMLメールのカスタマイズやバウンス（送信失敗）の管理など、メール運用に必要な機能が充実しています。CLF-C02 では主にメール送信サービスとして問われます。

---

> **試験のポイント**
> - Amazon Connect = クラウドベースのコンタクトセンター（コールセンター）
> - Amazon SES = 大量メール送信に特化したサービス
> - SES と SNS の違い: SES はメール専用、SNS は複数チャネルへの通知

---

## 16-4. バッチ処理とハイブリッド

### このセクションで学ぶこと

大規模なバッチ処理を実行する AWS Batch と、AWSのインフラをオンプレミス（自社のデータセンター）に持ち込む AWS Outposts を学びます。特に Outposts はハイブリッドクラウドの文脈で試験に出題されるため、しっかり押さえましょう。

### AWS Batch ― 大規模バッチジョブの実行

AWS Batch は、大量のバッチ処理ジョブ（まとめて実行する計算処理）を効率的に実行するためのサービスです。

バッチ処理とは、データや計算を一定量ためてから一気にまとめて処理する方式のことです。第12章で「バッチ処理は洗濯物をまとめて洗うようなもの」と説明しました。AWS Batch は、その「まとめ洗い」を大規模にやってくれる洗濯工場のような存在です。処理すべきジョブの数に応じて、必要なコンピューティングリソース（EC2 インスタンスや Fargate など）を自動的に確保し、ジョブが終わったらリソースを解放します。

科学技術計算、金融リスク分析、画像・動画のエンコード処理など、大量の計算を一気にこなしたい場面で使われます。オンプレミス環境でジョブスケジューラー（JP1 や Systemwalker など）を使ったことがある方は、それらのクラウド版と考えるとわかりやすいでしょう。ただし AWS Batch はリソースの確保まで自動化される点が異なります。自分でサーバーの台数を管理する必要がなく、ジョブを投入すればあとは AWS Batch がよしなにやってくれます。

### AWS Outposts ― AWSインフラをオンプレミスに設置

AWS Outposts は、AWSのハードウェア（サーバーやストレージ）を自社のデータセンターやオフィスに物理的に設置するサービスです。

これはなかなかユニークなサービスです。通常、AWSを使うときはインターネット経由でクラウドにアクセスしますが、Outposts では逆に「AWSの機器が自社に届いて、自社の中にAWSリージョンの小さな出張所ができる」イメージです。

なぜわざわざそんなことをするのでしょうか。理由はいくつかあります。法規制やコンプライアンスの都合上、データを自社の建物の外に出せないケース。工場や病院など、ネットワークの遅延（レイテンシー）を極限まで小さくしたいケース。既存のオンプレミス環境とクラウドをシームレスにつなぎたいケース。こうした場面で Outposts が力を発揮します。

Outposts には、サーバーラック単位で設置する Outposts Rack と、小型サーバー単位で設置する Outposts Server の2つの形態があります。規模やニーズに応じて選べます。

重要なのは、Outposts 上のリソースは通常のAWSと同じAPI（クラウド上のサービスに命令を送るための窓口のようなもの）で管理できるという点です。EC2 インスタンスを起動するのも、S3 にデータを保存するのも、クラウド上と同じ方法で行えます。つまり、場所はオンプレミスだけど、操作感はクラウドと同じです。

第5章で学んだ Direct Connect（専用線接続）や VPN と組み合わせることで、オンプレミスとクラウドを統合した「ハイブリッドクラウド」環境を構築できます。また、第14章の移行戦略で触れたように、すべてをクラウドに移行できない場合の受け皿としても Outposts は有効です。

試験では「ハイブリッドクラウド」「オンプレミスでAWSのサービスを使いたい」「データの所在地に関する規制がある」といったキーワードが出たら Outposts を思い浮かべてください。

---

> **試験のポイント**
> - AWS Batch = 大規模バッチジョブの自動実行。リソースの確保・解放を自動管理
> - AWS Outposts = AWSのハードウェアをオンプレミスに設置し、同じAPIで操作
> - Outposts は「ハイブリッドクラウド」の文脈で出題される
> - 「データを自社の外に出せない」「低レイテンシーが必要」→ Outposts

### 問題1

リモートワークを推進するために、社員にクラウド上の仮想デスクトップ環境を提供したい場合、最も適したサービスはどれですか？

- **A.** Amazon EC2
- **B.** Amazon WorkSpaces
- **C.** Amazon AppStream 2.0
- **D.** AWS Lambda

<details>
<summary>解答・解説</summary>

**解答: B**

解説: **Amazon WorkSpaces** は、クラウド上に仮想デスクトップ環境（DaaS）を提供するサービスです。リモートワーク用のデスクトップ環境として最適です。EC2はサーバー用途であり、直接操作するデスクトップ環境ではありません。AppStream 2.0はデスクトップ全体ではなく、特定のアプリケーションを配信するサービスです。

</details>

### 問題5

### 問題6

- **A.** Amazon SNS
- **B.** Amazon SQS
- **C.** Amazon SES
- **D.** Amazon Connect

<details>
<summary>解答・解説</summary>

**解答: C**

解説: **Amazon SES（Simple Email Service）** は、大量のメールを高い到達率で送信するためのサービスです。注文確認メールやキャンペーンメールなどの一括送信に適しています。SNSも通知の送信が可能ですが、メール送信に特化した機能（HTMLメールのカスタマイズ、バウンス管理など）はSESの方が充実しています。

</details>
