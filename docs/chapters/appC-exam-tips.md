# 付録C　試験対策のポイント

**対応試験ドメイン**: 全ドメインに横断的に対応
**推定読了時間**: 15分

---

## この付録の目的

AWS Certified Cloud Practitioner（CLF-C02）試験の合格に向けた、実践的なアドバイスをまとめています。学習リソースの使い方から試験当日の注意事項まで、合格のコツをお伝えします。

---

## C-1. 試験の全体像と戦略

### このセクションで学ぶこと

試験の形式・時間配分・基本戦略を確認します。

### 試験の基本情報

| 項目 | 内容 |
|------|------|
| 試験時間 | **90分** |
| 問題数 | **65問**（うち採点対象：50問、採点対象外（スコアリングなし）：15問） |
| 合格スコア | **700 / 1000** |
| 受験料 | **100 USD** |
| 問題形式 | 択一選択式（1つ選ぶ）、複数選択式（複数選ぶ） |

採点対象外の15問はAWSが新問題を検証するためのもので、どれが対象外かは受験中にはわかりません。65問すべてを真剣に解きましょう。

### 時間配分の目安

90分 ÷ 65問 ≒ 1問あたり**約80秒**

1問ずつ考えていると時間が足りなくなる可能性があります。以下の戦略を覚えておきましょう。

**推奨する時間配分戦略**

1. **1周目（約60分）**: 全問題を解く。確信がない問題は「後で見直す」にチェックして先に進む
2. **2周目（約25分）**: チェックをつけた問題を見直す
3. **最終確認（約5分）**: 未回答がないか確認

### 解答のコツ

- **消去法を使う**: 明らかに違う選択肢を2つ消せれば、残りは2択になる
- **わからない問題は後回し**: 悩んでいる間に時間が過ぎる
- **直感を信じる**: 見直しで答えを変えると間違えることが多い

---

> **ポイント**
> - 1問あたり約80秒が目安
> - わからない問題は印をつけて後回し
> - 採点対象外の15問があるが、どれかわからないので全問真剣に取り組む

| ドメイン | 配分 | 対応する主な章 |
|----------|------|--------------|
| **第1分野: クラウドのコンセプト** | 24% | 第0章、第14章、第15章（一部） |
| **第2分野: セキュリティとコンプライアンス** | 30% | 第9章、第0章（責任共有モデル）、第5章（一部） |
| **第3分野: クラウドテクノロジーとサービス** | 34% | 第1〜8章、第10〜13章、第16章 |
| **第4分野: 請求、料金、サポート** | 12% | 第15章、付録B |

> （draw.io ファイルは `docs/images/appC-domain-priorities.drawio.png` に配置予定）

### 章の学習優先度

限られた時間で効率的に学習するため、章を3つのグループに分類しています。

**必須章（合格ラインに直結・最優先）**

| 章 | タイトル | 理由 |
|----|---------|------|
| 第0章 | 導入（クラウドの基本概念） | 全ドメインの土台。IaaS/PaaS/SaaS、責任共有モデルは最頻出 |
| 第1章 | Webホスティング（EC2） | EC2購入オプション、ELB、Auto Scalingは頻出 |
| 第3章 | ストレージ（S3） | S3ストレージクラス、耐久性は頻出 |
| 第4章 | データベース（RDS/DynamoDB） | マルチAZ vs リードレプリカは超頻出のひっかけ |
| 第5章 | ネットワーク（VPC） | セキュリティグループ vs NACL は頻出 |
| 第9章 | セキュリティ（IAM） | 第2分野30%を占める最重要章 |
| 第10章 | 監視・運用（CloudWatch/CloudTrail） | CloudWatch vs CloudTrail の違いは頻出 |
| 第15章 | コスト管理 | 第4分野12%をカバー |
| 付録B | サポートプラン | 第4分野からの出題 |

**推奨章（スコアアップに有効）**

- 第2章（サーバーレス）、第6章（非同期処理）、第7章（耐障害性）
- 第8章（コンテナ）、第11章（CI/CD）、第14章（クラウド移行）
- 付録A（ネットワーク基礎）

**参考章（余裕があれば）**

- 第12章（データ分析）、第13章（AI/ML）、第16章（その他頻出サービス）

### 学習スタイルのアドバイス

**用語の暗記より概念の理解を優先する**

「S3の耐久性は99.999999999%（イレブンナイン）」のような数値は覚える価値があります。しかし試験の多くは「このシナリオでどのサービスを使うべきか」という理解を問う問題です。

**「なぜこのサービスを選ぶのか」を常に考える**

例: 「大量のデータを一時保存してから処理したい → SQS（メッセージキュー）を使う。なぜなら密結合を避けて疎結合にできるから」

> **図解:** appC-study-schedule.drawio.png — 2週間/1ヶ月/2ヶ月の学習スケジュールモデル
> （draw.io ファイルは `docs/images/appC-study-schedule.drawio.png` に配置予定）

**学習スケジュールの目安**

| 期間 | 推奨プラン |
|------|-----------|
| **2週間** | 必須章（第0・1・3・4・5・9・10・15章・付録B）に絞って集中学習。模擬試験1〜2回 |
| **1ヶ月** | 必須章 + 推奨章を学習。模擬試験3〜4回。弱点を集中的に復習 |
| **2ヶ月** | 全章を学習。模擬試験5回以上。余裕を持った復習 |

---

> **ポイント**
> - セキュリティ（30%）とテクノロジー（34%）で全体の64%を占める
> - 用語の暗記より「なぜそのサービスを使うか」の理解を優先
> - 必須章から始めて、徐々に範囲を広げる

## C-3. 模擬試験の活用法

### このセクションで学ぶこと

模擬試験を最大限に活用するための方法を学びます。

### 利用できる模擬試験

**AWSが提供するもの**

- **AWS Certification 公式練習問題集**: AWS Skill Builder（スキルビルダー）から無料で利用できます
- **AWS Certified Cloud Practitioner 公式模擬試験**: 有料（約20 USD）ですが、本番に近い問題形式で練習できます

**サードパーティの問題集**

- Udemy の試験対策コース（日本語対応のものもある）
- Whizlabs、Tutorials Dojo などの問題集サイト

### 模擬試験の活用法：「復習が本番」

模擬試験は「受けて終わり」では効果が半減します。以下の手順で復習することで学習効果を最大化できます。

1. **本番と同じ環境で解く**: タイマーを設定し、90分で65問を解く
2. **採点後、まず全問見直す**: 正解・不正解にかかわらず、全問の解説を読む
3. **不正解の原因を分析する**:
   - 「知識が不足していた」 → 該当章を読み直す
   - 「問題文を読み違えた」 → 問題文をゆっくり読む練習をする
   - 「迷ったが間違えた」 → 似たような問題を追加で解く
4. **弱点を記録する**: 間違えた問題のカテゴリをメモし、重点的に復習する

### 模擬試験のスコア目標

| スコア | 判断 |
|--------|------|
| 700以上 | 本番受験を検討してよい |
| 600〜699 | あと少し。弱点分野を集中復習 |
| 600未満 | 基礎から見直す |

---

> **ポイント**
> - 模擬試験は「復習が本番」。解説をしっかり読む
> - 不正解の原因を「知識不足」「読み違い」「迷い」の3種類に分類する
> - 模擬試験700点以上を目安に本番受験を検討

## C-4. よく出るパターンと対策

### このセクションで学ぶこと

試験で頻繁に出る問題パターンを把握し、解答速度を上げます。

### パターン1: 「最適なサービスはどれか？」型

シナリオが与えられ、最も適切なAWSサービスを選ぶ問題です。最も出題数が多いパターンです。

**解き方のコツ**

- キーワードを拾う：「サーバーレス」「マネージド」「高可用性」「コスト最適化」
- 消去法：明らかに違うサービスを先に外す

**例題1**

> ある企業は、画像ファイルをアップロードするたびに自動的にサムネイルを生成したいと考えています。サーバーの管理は行いたくありません。最も適切なサービスはどれですか？
>
> A. Amazon EC2
> B. AWS Lambda
> C. Amazon ECS
> D. AWS Fargate

**解答: B（AWS Lambda）**

解説: 「サーバーの管理不要」「イベント駆動（アップロードをトリガーに実行）」がキーワードです。Lambdaはサーバーレスでイベント（S3へのアップロードなど）をトリガーに関数を実行できます。

---

**例題2**

> 大量のユーザーが同時にアクセスしても安定して動作するWebアプリを構築したいと考えています。トラフィックに応じて自動的にサーバー数を調整したいです。最も適切な組み合わせはどれですか？
>
> A. EC2 + Elastic Load Balancing + Auto Scaling
> B. EC2 + CloudFront + Route 53
> C. Lambda + API Gateway + DynamoDB
> D. ECS + Fargate + CloudFormation

<details>
<summary>解答・解説</summary>

**解答: A**

解説: 「トラフィックに応じて自動調整」= Auto Scaling、「複数サーバーへの振り分け」= ELBの組み合わせです。

</details>

---

### パターン2: 「責任共有モデルでは誰の責任か？」型

AWSと顧客の責任範囲を問う問題です。

**解き方のコツ**

- 物理インフラ（ハードウェア、データセンター）→ **AWSの責任**
- データ、アクセス管理、暗号化設定 → **顧客の責任**
- マネージドサービスのOS（RDS、Lambdaなど）→ **AWSの責任**
- EC2のOS、ミドルウェア → **顧客の責任**

**例題3**

> Amazon RDS を使用しているとき、**顧客の責任**はどれですか？
>
> A. データベースエンジンのパッチ適用
> B. データベースへのアクセス権限の管理
> C. 物理サーバーのハードウェア障害への対応
> D. 仮想化ソフトウェアの管理

<details>
<summary>解答・解説</summary>

**解答: B**

解説: RDSはマネージドサービスなので、OSパッチ・ハードウェア・仮想化はAWSの責任です。データベースへの「誰がアクセスできるか」の管理は顧客の責任です。

</details>

---

### パターン3: 「コスト最適化のために何をすべきか？」型

コスト削減の方法を問う問題です。

**解き方のコツ**

- 長期利用が確定している → リザーブドインスタンス or Savings Plans
- 中断してもいい処理 → スポットインスタンス
- 未使用リソースの検出 → Trusted Advisor / Cost Explorer
- ストレージコスト削減 → S3ライフサイクルポリシーで適切なストレージクラスに移行

**例題4**

> ある企業は、今後3年間継続して使用するデータベースサーバーのコストを最適化したいと考えています。最も効果的な方法はどれですか？
>
> A. スポットインスタンスを使用する
> B. オンデマンドインスタンスを使用する
> C. 3年リザーブドインスタンスを購入する
> D. Dedicated Hosts を使用する

<details>
<summary>解答・解説</summary>

**解答: C**

解説: 「3年間継続使用が確定」しているため、3年リザーブドインスタンスが最もコスト削減効果が高いです。スポットインスタンスは中断の可能性があるためDBサーバーには不向きです。

</details>

---

> **ポイント**
> - 問題文のキーワードを拾う（「サーバーレス」「マネージド」「自動スケーリング」など）
> - 責任共有モデルは「物理インフラはAWS」「データ・アクセス管理は顧客」が基本
> - コスト問題は「期間の長さ」「中断可能か」で選択肢を絞る

## C-5. 試験当日の注意事項

### このセクションで学ぶこと

テストセンター受験・オンライン受験それぞれの注意事項と、直前の準備を確認します。

### テストセンター受験

- **身分証明書**: 有効期限内の身分証明書が2点必要です。1点目は政府発行の顔写真付きID（例: 運転免許証、パスポート）、2点目は署名付きID（例: クレジットカード）。詳細はPearson VUEのサイトで最新の要件を確認してください
- **持ち込み禁止物**: スマートフォン、メモ帳、参考書は持ち込み禁止。ロッカーに預けます
- **早めに到着**: 受験開始の30分前には到着するのが安心です
- **メモ用紙**: 試験センターから消えるペン・ホワイトボードが提供されます

### オンライン受験（自宅・会社など）

- **部屋の準備**: 個室で、他の人が入ってこない環境が必要です
- **デスクの整理**: 机の上には何も置いてはいけません（水・飲み物も原則不可）
- **カメラ・マイク**: 動作確認を事前に行いましょう
- **プロクタリングソフト**: 試験前に監視ソフトをインストールする必要があります

### 当日の直前確認ポイント

試験前夜・当日朝に素早く確認すると効果的な項目です。

**第2分野（セキュリティ）の最終確認**

- 責任共有モデル（AWSの責任 vs 顧客の責任）
- IAMのユーザー・グループ・ロール・ポリシーの関係
- Shield Standard（無料、全アカウントに自動適用）

**よく出るサービスの最終確認**

- S3ストレージクラス（Standard → IA → Glacier の順にコスト安）
- RDSのマルチAZ（高可用性）vs リードレプリカ（読み取りパフォーマンス）
- CloudWatch（監視）vs CloudTrail（監査ログ）vs Config（構成管理）
- サポートプランの応答時間（特に Enterprise の15分）

---

> **ポイント**
> - 試験は90分で65問。時間配分を意識する
> - テストセンターは30分前到着を目安に
> - 直前は覚えにくかった概念の最終確認に集中する（新しいことを詰め込まない）

### 問題1

クラウドコンピューティングの利点として、正しいものはどれですか？

- **A.** 初期費用（資本的支出）が大きくかかる
- **B.** リソースの調達に数ヶ月かかる
- **C.** 需要に応じてリソースを柔軟に増減できる
- **D.** 利用料金は一律の固定月額制である

<details>
<summary>解答・解説</summary>

**解答: C**

クラウドの「弾力性（Elasticity）」です。初期費用最小化・即時調達・従量課金もクラウドの利点です。

</details>

---

### 問題2

Amazon EC2 インスタンスの購入オプションとして、**中断されることがあるが最もコストが低い**ものはどれですか？

- **A.** オンデマンドインスタンス
- **B.** リザーブドインスタンス
- **C.** スポットインスタンス
- **D.** Dedicated Hosts

<details>
<summary>解答・解説</summary>

**解答: C**

スポットインスタンスは未使用のEC2容量を使うため安価ですが、AWSが必要とした際に中断されることがあります。

</details>

---

### 問題3

AWS の責任共有モデルにおいて、**顧客の責任**はどれですか？

- **A.** データセンターの物理的なセキュリティ
- **B.** ハイパーバイザー（仮想化ソフトウェア）の管理
- **C.** S3バケットに保存されたデータの暗号化
- **D.** ネットワーク機器のメンテナンス

<details>
<summary>解答・解説</summary>

**解答: C**

データの暗号化とアクセス管理は常に顧客の責任です。物理インフラ・ハイパーバイザーはAWSの責任です。

</details>

---

### 問題4

Amazon S3 のストレージクラスの中で、**アクセス頻度が低く、取り出しに数時間かかるが最もコストが低い**ものはどれですか？

- **A.** S3 Standard
- **B.** S3 Standard-IA
- **C.** S3 Glacier Deep Archive
- **D.** S3 One Zone-IA

<details>
<summary>解答・解説</summary>

**解答: C**

Glacier Deep Archive は最も安価なストレージクラスで、取り出しに最大12時間かかります。長期間アーカイブに適しています。

</details>

---

### 問題5

Amazon RDS の「マルチAZデプロイ」の主な目的はどれですか？

- **A.** 読み取りパフォーマンスの向上
- **B.** 複数のリージョンへのデータレプリケーション
- **C.** 障害発生時の自動フェイルオーバーによる高可用性の確保
- **D.** データベースのコスト削減

<details>
<summary>解答・解説</summary>

**解答: C**

マルチAZは高可用性（HA）のための機能です。同一リージョン内の別AZにスタンバイを作り、障害時に自動でフェイルオーバーします。読み取りパフォーマンス向上はリードレプリカの目的です。

</details>

---

### 問題6

Amazon VPC のパブリックサブネットとプライベートサブネットの違いとして、正しいものはどれですか？

- **A.** パブリックサブネットにはインターネットゲートウェイへのルートがある
- **B.** プライベートサブネットからはインターネットに一切アクセスできない
- **C.** パブリックサブネットのEC2インスタンスはファイアウォールが不要
- **D.** プライベートサブネットはパブリックより高速である

<details>
<summary>解答・解説</summary>

**解答: A**

パブリックサブネットはルートテーブルにインターネットゲートウェイへのルートが設定されており、インターネットと直接通信できます。B は誤りです。プライベートサブネットからも NATゲートウェイ経由でインターネットにアクセスできます。

</details>

---

### 問題7

AWS IAMのロール（Role）の説明として、正しいものはどれですか？

- **A.** 特定のAWSユーザーに永続的に割り当てられた権限のセット
- **B.** 一時的な認証情報を提供し、AWSサービスや外部エンティティに権限を付与する仕組み
- **C.** 複数のユーザーをまとめてグループ化する機能
- **D.** AWSアカウントの管理者用の特別なユーザー

<details>
<summary>解答・解説</summary>

**解答: B**

IAMロールは「一時的な認証情報を使って権限を付与する」仕組みです。EC2インスタンスに権限を付与したり、外部ユーザーが一時的にAWSにアクセスする際に使います。

</details>

---

### 問題8

Amazon CloudWatch と AWS CloudTrail の違いとして、正しいものはどれですか？

- **A.** CloudWatch は API 操作の監査ログを記録し、CloudTrail はメトリクスを監視する
- **B.** CloudWatch はリソースのメトリクス・ログを監視し、CloudTrail は API 操作の監査ログを記録する
- **C.** CloudWatch と CloudTrail は同じ機能を持つ別名
- **D.** CloudWatch はセキュリティ、CloudTrail はパフォーマンスを担当する

<details>
<summary>解答・解説</summary>

### 問題14
