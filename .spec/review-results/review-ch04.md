#### 良い点
- 「データの図書館」「ロッカーの番号札」「住みながらの引っ越し」「災害時の非常口」「窓口の増設」など、身近なアナロジーが豊富で、初心者の理解を大きく助けている。
- EC2に自前MySQLをインストールする場合とRDSの比較表が、マネージドサービスのメリットを具体的に示しており分かりやすい。
- 各セクション冒頭の「このセクションで学ぶこと」が学習の見通しを立てやすくしている。
- 章末問題が本文の重要ポイントを網羅的にカバーしており、学習の定着に有効。
- 試験のポイントが各セクション末にまとめられており、復習に使いやすい。

#### 良い点
- マルチAZ vs リードレプリカの比較表は、オンプレミスでのHA構成やリードスレーブとの対比で理解しやすい。
- DMS の CDC（Change Data Capture）への言及があり、移行の実務経験者にとって重要なポイントをカバーしている。
- DMS + SCT の使い分け（同種間/異種間）の整理が明確。
- Aurora のストレージアーキテクチャ（3AZ x 6コピー、自動拡張）の記載は正確。
- 章末問題に「2つ選択」形式（問題4）が含まれており、実際のCLF-C02試験の出題形式を反映している。

## 横断的サマリー（全ロール共通）

### 優先修正事項（全ロールに影響）
1. **QLDB のサービス終了への対応**（R2-002）: QLDBは2025年7月に終了済み。試験対策資料としての信頼性に影響するため、注記の追加または代替サービスへの言及が必要。
2. **リードレプリカの最大数の不正確さ**（R2-001）: Auroraを含む文脈で「最大5つ」は誤り。全ロールに影響する事実誤認。
3. **同期/非同期レプリケーションの説明補足**（R1-003）: 両方の用語が頻出するが、違いの明示的な説明がない。

### ロール固有の主要課題
- **IT初心者**: 専門用語（DNS、ストアドプロシージャ、CDC、パーティションキー等）に対する平易な説明の不足。アナロジーは豊富だが、用語の初出時の定義がやや甘い。
- **ベテランエンジニア**: RDS Proxy、DynamoDBキャパシティモード、Aurora Global Database など、実務・試験の両面で重要な概念の欠落。QLDB廃止という最新情報への未対応が信頼性を下げる。

### 修正後の確認ポイント
- [ ] リードレプリカの最大数がAurora（15個）とそれ以外（5個）で区別されているか
- [ ] QLDBの説明にサービス終了の注記が追加されているか
- [ ] 「同期レプリケーション」と「非同期レプリケーション」の違いが平易に説明されているか
- [ ] DNS、ストアドプロシージャ、CDC等の専門用語に初心者向けの補足があるか
- [ ] RDS Proxy について少なくとも1段落の説明が追加されているか
- [ ] DynamoDB のキャパシティモード（オンデマンド/プロビジョンド）に言及があるか
- [ ] RDS/Aurora のスケーリング説明がリードレプリカを考慮した記載になっているか
- [ ] 全体の試験ポイントが CLF-C02 の第3分野（クラウドテクノロジーとサービス）に適合しているか
