# learning スキル

## 目的

- ドキュメントをchapter毎に読み込んで、ユーザに対して授業を行う

## ワークフロー

1. ユーザに対して、テストか、授業かを確認
  - テストの場合、2へ
  - 授業の場合、3へ

2. テストの希望を聞いて、要件に従って tools/generate_exam.py を実行し、テストを生成

3. ユーザに対して、過去の続きからか、指定した場所から授業するかを確認
  - 過去の続きからの場合、records ディレクトリの記録ファイルをチェックし、進捗を把握
  - ユーザ指定の場合、ユーザに希望を訪ねる

4. docs/chapters 配下のファイルを読み込んで、授業をスタート

## 授業の要件

- 記録が残っていなければ、chapter ファイルの頭から解説していく
- 適宜、ユーザに対して質問し、理解度を確認する
- 授業が完了したら、ユーザに対して内容をフィードバックする
  - 必ずポジティブな内容から始めつつ、必要に応じて理解度を深める為のアドバイスをする
- records 配下に、授業の内容を記録する
  - 授業のサマリ
  - ユーザの理解度
  - アドバイスの内容
  - 理解を深める為の推奨事項
